# Radius — Roadmap

## Phase 1: Foundation (Q2 2026)
- [RC] WebAssembly simulation kernel — Rust-based, SIMD-enabled
- [RC] Radius DSL v1 — declarative physics definition language
- [RC] TypeScript SDK — programmatic simulation control
- [RC] Browser renderer — WebGPU 3D visualization
- [RC] Landing page live at radius.blackroad.io
- [RC] Physics domain: rigid body dynamics, gravity, constraints

## Phase 2: Multi-Domain (Q3 2026)
- [RC] Chemistry domain: molecular dynamics, reaction kinetics
- [RC] Biology domain: protein folding, population dynamics
- [RC] Earth domain: basic weather and ocean models
- [RC] FEA module for structural analysis
- [RC] CFD solver for fluid simulations
- [RC] Data export: CSV, JSON, HDF5
- [RC] CAD import: STEP, STL, OBJ file support

## Phase 3: Collaboration & Sharing (Q4 2026)
- [RC] CRDT real-time collaboration engine
- [RC] Shareable simulation URLs with version history
- [RC] Fork / modify / merge simulation workflows
- [RC] Team workspaces with role-based access
- [RC] DOI citation integration for academic use
- [RC] Community plugin marketplace launch

## Phase 4: Enterprise & Digital Twins (Q1 2027)
- [RC] Self-hosted enterprise deployment
- [RC] Air-gapped mode (ITAR/EAR compliance)
- [RC] Digital twin export and IoT sensor integration
- [RC] Edge device deployment (Raspberry Pi, Hailo-8)
- [RC] SSO / SAML authentication
- [RC] SLA and dedicated support tiers

## Phase 5: Scale & Ecosystem (Q2 2027)
- [RC] Plugin SDK for Rust, C++, AssemblyScript
- [RC] GPU compute cluster for large-scale simulations
- [RC] Jupyter notebook integration
- [RC] VS Code extension for Radius DSL
- [RC] Academic institution licensing program
- [RC] 1,000+ community plugins target

## Long-Term Vision
- [RC] Become the GitHub for physical simulation
- [RC] Every university student has access to world-class simulation tools
- [RC] Bridge the gap between computational research and physical engineering
- [RC] Sovereign compute — no data leaves the user's control unless they choose
