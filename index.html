Write Simulations
Like Code

Radius uses a declarative simulation language that feels familiar to any developer. Define your system, set initial conditions, and let the engine handle the numerics. Full TypeScript SDK for programmatic control.

  • Declarative physics DSL with type inference
  • Hot-reload — change parameters without restarting
  • Built-in unit system (SI, CGS, Imperial)
  • Automatic differentiation for sensitivity analysis
  • GPU compute shaders for particle systems
// Radius simulation — pendulum system
import { Simulation, Gravity, Constraint } from '@radius/core';

const sim = new Simulation({
  timestep: 0.001,       // seconds
  duration: 30,           // seconds
  renderer: 'webgpu',
});

// Add forces
sim.addForce(new Gravity({ g: 9.81 }));

// Define bodies
const bob = sim.addBody({
  mass: 1.0,              // kg
  position: [2.0, 0.0],  // meters
  radius: 0.05,
  material: 'steel',
});

// Pin constraint
sim.addConstraint(new Constraint.Rod({
  bodyA: 'origin',
  bodyB: bob,
  length: 2.0,
}));

// Run and observe
sim.onFrame((state) => {
  console.log(state.energy);
});

sim.run();

How Radius Works

A four-layer architecture designed for performance, extensibility, and sovereignty.

01

Model Definition

Define your system using the Radius DSL or TypeScript SDK. Declare bodies, forces, constraints, and boundary conditions.

02

Compilation

The Radius compiler optimizes your model and emits a .wasm binary with specialized numerical solvers for your system type.

03

Execution

Run the simulation in your browser using WebAssembly with SIMD and threading support. GPU compute via WebGPU for particles.

04

Analysis

Real-time visualization, data export, sensitivity analysis, and digital twin deployment to edge devices or cloud.
